# CyanCruise 简历面试知识库

## 简历结构

计算机相关岗位简历通常包含基本信息、求职意向、教育经历、专业技能、项目经历、实习经历、竞赛或证书。学生项目应突出业务场景、个人职责、技术方案和可验证结果。

## 项目描述方式

推荐使用“背景-任务-行动-结果”的经历讲述框架，但面向用户时应称为“经历讲述框架”，避免直接使用专业缩写。

示例结构：

- 项目背景：为什么做这个系统，服务什么用户或业务。
- 个人职责：自己负责哪些模块、接口、页面、数据库表或部署工作。
- 技术方案：使用了哪些技术解决什么问题。
- 难点处理：权限、并发、性能、数据一致性、异常处理、可维护性等。
- 结果复盘：功能完成度、性能改善、用户反馈、可演示地址或代码仓库。

## 校园招聘系统项目示例

原始表达：做了一个校园招聘系统，使用 Java 和 MySQL。

优化表达：负责校园招聘系统的岗位发布、简历投递和面试进度模块，基于 Java 后端实现岗位查询、投递记录、状态流转和权限校验接口；设计岗位表、投递表和面试记录表，支持学生按岗位类别和城市筛选，并为企业端提供投递列表和状态更新能力。

## 面试准备

- 项目面试：准备系统架构图、核心表结构、接口清单和关键流程。
- 技术面试：围绕 Java 集合、线程、数据库索引、事务、HTTP、缓存和框架基础复习。
- 行为面试：准备学习能力、协作、压力处理、项目复盘和职业规划案例。
- 反问环节：可询问团队业务、技术栈、培养机制和岗位发展。

## 输出要求

简历优化建议应直接给出可替换的中文描述，并解释为什么这样写更清楚。面试建议应包含准备清单和练习问题。
