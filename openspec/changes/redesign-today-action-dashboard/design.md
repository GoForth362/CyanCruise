## Context

今日行动页由 `renderTodayPage` 组合每日卡、本周卡和路线来源卡。任务勾选分别通过 `data-daily-task` 与 `data-plan-task` 进入既有事件委托。当前两张任务卡并排且信息等权，长任务文本会放大页面宽度和阅读负担；页面也没有直接展示今日完成比例与顺延任务数量。

## Goals / Non-Goals

**Goals:**

- 使用真实每日任务状态展示完成数、剩余数和顺延数量。
- 强化今日任务优先级，将本周计划和路线来源作为辅助信息。
- 消除横向滚动并提升长文本、窄屏和键盘操作体验。
- 保留现有任务数据属性、状态持久化和路线隔离。

**Non-Goals:**

- 不改变每日任务生成、顺延、勾选接口或任务 ID。
- 不把本周任务计入路线总进度。
- 不修改完整规划页或首页今日行动摘要。
- 不新增后端接口和依赖。

## Decisions

### 使用真实任务数组计算今日概览

从 `view.dailyPlan.items` 读取完成状态和 `carriedOver` 标记，计算今日完成数、剩余数、顺延数与百分比。对于本地派生任务，沿用 `progressState.checked` 判断，不引入演示数据。

### 采用主次分栏而不是等权清单

桌面端今日任务占主要宽度，本周节奏作为右侧辅助卡；较窄桌面和移动端统一变为单列。概览独立置顶，用户无需扫描全部清单即可理解当前状态。

### 保持原任务渲染契约

继续通过 `renderLinkedTaskList`、`renderTaskList` 生成复选框，保留 `data-daily-task`、`data-plan-task`、`data-source-task` 和任务 ID。仅增加结构类与视觉包装。

### 使用专属页面模式隔离样式

仅在 `today-action` 路由添加 `today-action-dashboard-mode`，强化页头、任务卡和来源区域，不影响完整规划页复用的任务样式。

## Risks / Trade-offs

- [本地派生任务完成状态计算不一致] → 抽取与现有渲染相同的判断方法，持久化任务优先使用 `item.completed`。
- [长文本撑开网格] → 所有网格子项使用 `min-width: 0`，任务文本允许任意位置换行。
- [异步勾选重绘导致进度滞后] → 概览每次 `renderPage` 都从当前状态重新计算。
- [窄屏信息过长] → 在 1080px 以下切为单列，按钮与元信息允许换行。

## Migration Plan

1. 增加页面状态类和真实进度计算。
2. 重构今日概览、每日、本周与路线来源结构。
3. 增加专属响应式、焦点和减少动态效果样式。
4. 更新静态契约和缓存版本，完成视觉及构建验证。

回滚只涉及前端结构、样式与缓存版本，不涉及数据迁移。
