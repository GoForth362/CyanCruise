## Context

路径规划页由 `renderCareerPlanPage` 组合规划依据、每日计划、本周计划、总进度、箭头阶段流和阶段详情。阶段任务通过 `data-plan-task` 持久化，阶段选择通过 `data-plan-focus` 更新视图。现有结构功能完整，但所有模块等权连续排列；规划依据行过宽，阶段流依赖横向排列，容易在平台容器中产生横向滚动。

## Goals / Non-Goals

**Goals:**

- 用真实路线状态建立清晰的总览、里程碑、执行和阶段层级。
- 保留就业与升学目标语义及所有任务、阶段操作。
- 消除横向滚动，优化长文本和窄屏体验。
- 提供明确的键盘焦点和减少动态效果适配。

**Non-Goals:**

- 不改变规划生成、刷新保护、阶段进度和每日任务接口。
- 不改变一年规划筛选和服务端阶段顺序。
- 不重构就业/升学中心和今日行动页。
- 不新增后端接口或依赖。

## Decisions

### 添加专属路径规划页面模式

仅在 `career-plan` 路由添加 `career-plan-dashboard-mode`，新样式限定在该页面，避免影响今日行动或中心页复用的任务、路线组件。

### 将真实路线指标置于总览

从 `view.targetRole`、`view.progressSummary`、`view.visiblePhases` 和 `view.selectedYears` 展示目标、当前阶段、阶段数量、任务完成数和总进度，不生成演示指标。
升学路线同时读取当前用户已保存的目标院校；若真实升学计划尚未生成，本周计划只展示等待生成状态，不回退到简历、岗位和模拟面试任务。

### 保留现有组件并增加专属包装

继续调用 `renderDailyPlanCard`、`renderWeeklyPlanCard`、`renderPlanTimeline`、`renderPlanFlow` 和 `renderPlanPhaseCard`，通过新增包装和专属类调整页面层级。这能最大限度保留任务 ID、阶段焦点和勾选逻辑。

### 阶段导航在窄屏改为可换行网格

桌面端阶段导航保持顺序感，窄屏改为单列或双列卡片，不依赖横向滚动。连接箭头在窄屏隐藏，顺序由编号和 DOM 顺序表达。

## Risks / Trade-offs

- [共享路线组件被新样式影响] → 所有增强样式限定在 `career-plan-dashboard-mode` 下。
- [空规划状态缺少总览数据] → 保留原空状态与生成入口，总览只使用可用真实值。
- [阶段数量超过四个] → 导航使用可换行网格，阶段详情继续完整展示。
- [长任务或依据文本撑开页面] → 网格子项设置 `min-width: 0` 并允许任意位置换行。

## Migration Plan

1. 增加页面状态类和真实路线总览。
2. 重排依据、执行计划、里程碑与阶段区域。
3. 增加专属响应式与可访问性样式。
4. 更新静态契约和资源版本，完成视觉及构建验证。

回滚只涉及前端结构、样式和缓存版本，不涉及数据迁移。
