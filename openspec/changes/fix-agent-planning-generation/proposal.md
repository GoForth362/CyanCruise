# 修复智能规划生成链路

## Why

升学规划与求职规划页面能够正常提交请求，但当前运行配置优先调用对话智能体。部分对话智能体只返回“准备调用任务流”的思考或工具调用过程，没有返回最终结构化路线 JSON，导致后端将有效请求判定为“规划结果格式不完整”。界面改版后该问题更容易暴露，用户即使填写了完整信息也无法生成规划。

## What Changes

- 允许升学规划按运行时配置优先直连已发布任务流，智能体编码仅作为兼容回退。
- 统一解析任务流和智能体的常见输出包装，提取真正满足路线契约的最终 JSON，忽略思考、工具调用参数和用户输入回显。
- 求职规划复用同一套结构化结果识别规则，避免出现相同的格式误判。
- 保留现有路线与完成进度；外部智能服务未产生合格结果时不保存虚假规划。
- 增加生成链路测试与可诊断日志。

## Capabilities

### New Capabilities

无。

### Modified Capabilities

- `study-route-ai-planning`：明确优先使用已配置任务流，并规范化平台包装结果。
- `employment-route-planning`：统一智能规划结果提取、校验和失败保护。
- `postgraduate-agent-route-planning`：补充对工具调用过程回显的过滤及任务流优先行为。

## Impact

- 后端智能规划客户端、运行时配置加载、共享路线 JSON 解析器。
- 升学规划与求职规划生成接口的错误诊断和自动化测试。
- 不改变现有前端接口契约、数据库结构和已保存路线数据。
