## Context

升学中心与就业中心都由 `app-runtime.js` 拼接页面结构，并复用路线图、区块、表单和资源卡片样式。现有能力完整，但首屏同时呈现标题、操作、表单、摘要和阶段卡，视觉优先级接近；后续洞察、资料与资讯区也缺少统一的分组语言。两个中心已有独立状态和接口，本次不需要新增服务端能力。

## Goals / Non-Goals

**Goals:**

- 为两个中心建立一致的页面骨架、青色视觉语言和状态表达。
- 用真实的方向、目标、规划与资料状态形成明确的首屏概览。
- 保留所有现有数据属性、接口调用和业务隔离。
- 支持桌面、窄屏、键盘焦点和减少动态效果偏好。

**Non-Goals:**

- 不改变升学方向保存、规划生成、阶段保护和资料解析逻辑。
- 不改变就业洞察、简历、面试和资源接口。
- 不把升学与就业的数据、资源或规划合并。
- 不重构完整规划页、今日行动页和各陪伴子页面。

## Decisions

### 使用共同模式与独立页面状态类

在两个中心首页分别添加 `study-center-dashboard-mode` 与 `employment-center-dashboard-mode`，共享 `path-center-*` 结构样式。相比覆盖通用 `.roadmap-*`，这种方式能统一观感又避免影响完整规划页和其他路线组件。

### 首屏采用概览与控制分层

就业中心首屏突出目标岗位和规划状态，升学中心首屏突出当前方向、目标院校与方向选择。主要操作保留在概览附近，但表单控件与路线摘要分开，降低信息拥挤。

### 复用真实状态，不推导虚假进度

方向、目标、阶段、资源数量和资料数量继续来自现有状态对象。页面只调整呈现方式，不新增演示百分比或虚构统计。

### 保留原事件契约

继续使用 `data-link="career-plan"`、`data-ensure-plan`、`data-ensure-study-plan`、`data-save-study-direction` 以及既有资料/资源数据属性，避免修改事件委托和服务调用。

### 返回时同步恢复滚动位置

资源页继续复用 `data-back-route` 和既有父路由映射，但目标中心完成同步渲染后立即恢复离开前的滚动位置，不再通过下一轮定时任务延迟滚动，避免浏览器先绘制中心页顶部再跳转。

## Risks / Trade-offs

- [共享样式影响其他页面] → 所有强化样式限定在中心页面状态类或 `path-center-*` 专属结构下。
- [异步数据重绘造成布局抖动] → 使用稳定网格和最小高度，加载、空状态继续复用原状态组件。
- [升学表单在窄屏过宽] → 控制区在窄屏变为单列，操作按钮可换行。
- [资源内容长度不一致] → 卡片摘要限制行数，完整内容仍从原详情入口查看。

## Migration Plan

1. 增加两个中心的页面状态类和专属结构。
2. 重构两个路线图首屏，保留全部事件数据属性。
3. 增加共享与分端样式、响应式和可访问性规则。
4. 扩展静态契约，更新资源版本并完成页面截图和构建验证。

回滚只需撤销对应结构、专属样式与缓存版本，不涉及数据迁移。
