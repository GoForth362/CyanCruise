## Context

CyanCruise 在迁移期为了让尚未接通的页面可操作，引入了前端预览数据和后端规则兜底生成器。随着真实接口、持久化与智能体逐步接入，这些机制仍可能在请求失败或空数据时生成完整题库、推荐、规划、任务或记录，使“暂无真实数据”与“已有业务结果”无法区分。本次变更跨越静态 webapp、应用服务和 AI 边界，必须以统一原则清理。

## Goals / Non-Goals

**Goals:**

- 建立“业务结果必须可追溯到真实接口、真实存储或用户输入”的统一规则。
- 删除运行期虚假数据生成器以及所有调用点。
- 失败时保留真实旧数据或用户草稿，并展示明确空态、不可用状态和重试操作。
- 通过静态扫描、单元测试、浏览器验收和 JDK 8 构建阻止回归。

**Non-Goals:**

- 不删除输入框 placeholder、格式示例、合法枚举标签和空集合初始化。
- 不删除测试源码中的测试夹具，也不把测试数据发布到运行期。
- 不删除无摄像头面试模式、配置默认超时或纯展示状态标签。
- 不清理已经持久化的数据；既有数据真实性需要另行审计。

## Decisions

1. 以“是否冒充用户业务结果”作为清理标准，而不是按变量名中的 `fallback`、`preview` 一刀切。替代方案是删除所有 fallback，但会破坏错误恢复、设备兼容和必要配置默认值。
2. 前端接口失败统一进入 unavailable；接口成功且空数据进入 empty。不得在 catch、身份缺失或空数组分支构造业务 DTO。
3. 本地草稿只保存用户输入字段，不得生成 ID、完成状态、分数、推荐或阶段；正式列表只接受服务端返回记录。
4. AI 与智能体失败返回明确错误，不再调用规则生成器产生问题、规划、诊断或报告。结构化响应无效时也不得保存部分或替代结果。
5. 职业计划的“确保计划”调整为“读取已有真实计划；没有则返回缺失摘要”。生成必须走明确的真实生成或用户保存动作。
6. 新增源代码静态契约，禁止已识别的运行期生成器名称、`preview` 业务 ID、规则兜底成功状态和示例资源回流。
7. 资源是否对用户展示以管理后台真实存储及 `hidden` 状态为准，不按历史资源 ID 建立永久黑名单；阻止运行期造数据与展示管理员已维护内容是两个独立边界。
8. 产品明确要求恢复的升学资讯作为“已发布内容目录”在升学中心应用初始化且存储为空时写入真实存储；它与查询失败时动态生成的兜底数据不同。初始化完成后写入持久化版本标记，已有管理员内容或已有标记时跳过恢复，避免覆盖或复活管理员治理结果。

## Risks / Trade-offs

- [风险] 接口未部署时页面内容会明显减少 → 使用清晰空态、错误原因和重试/创建入口表达真实状态。
- [风险] 旧流程依赖 ensure 自动造计划 → 调整调用方按 `hasPlan=false` 展示生成入口，并补充测试。
- [风险] AI 故障时面试无法继续 → 明确提示本次问题未生成，允许重试或结束，不保存无意义会话。
- [风险] 清理范围过大误删正常模板 → 静态扫描结果逐项分类，保留输入示例、空集合、测试夹具和用户草稿。

## Migration Plan

1. 建立虚假运行期数据清单并为每个生成器记录调用点。
2. 先移除前端 preview/default 业务数据，再调整后端规则兜底返回。
3. 更新调用方空态、不可用状态和契约校验。
4. 运行前端语法、OpenSpec、单元测试、浏览器场景和 JDK 8 全量构建。
5. 如需回滚，按功能模块恢复接口状态处理，但不得恢复伪造数据生成器。

## Open Questions

- 已持久化记录不按 ID 推断真实性。需要清理的 `RULE_FALLBACK`、`preview` 或 `fallback` 来源记录应通过可审计的数据迁移处理；管理员保留并设置为展示中的内容继续作为真实存储内容发布。
