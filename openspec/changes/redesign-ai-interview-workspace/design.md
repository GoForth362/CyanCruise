## Context

`interview` 页面由 `renderInterviewPage` 根据准备、进行中、分析中、分析失败、报告和问答记录状态选择渲染函数。准备页 `renderInterviewSetup` 当前把岗位、简历、难度和操作放在一个通用面板中；进行中与报告已有独立结构，但缺少与准备页一致的工作台视觉和状态表达。

## Goals / Non-Goals

**Goals:**

- 用真实岗位草稿、已选简历、难度和简历列表展示面试准备状态。
- 将准备页重构为概览、设置步骤和明确的开始操作。
- 统一准备中、面试房间、分析中、失败和报告的页面容器与青色视觉。
- 保留全部 DOM 标识、数据属性、草稿状态、语音输入、回答和结束操作。
- 消除窄屏横向滚动，支持长问题、长回答和键盘访问。

**Non-Goals:**

- 不修改面试问题数量、问题生成、评分算法、智能体提示词或历史保存。
- 不替换语音音色，不新增摄像头或全景面试功能。
- 不伪造准备完成、面试进度、分数或报告。
- 不修改面试服务、DTO、数据库或身份边界。

## Decisions

### 使用独立文本面试页面模式

仅在 `interview` 路由增加 `ai-interview-workspace-mode`。新样式限定在该模式下，避免影响全景仿真面试和面试历史页。

### 准备总览只读取真实草稿和简历状态

从 `interviewSetupPosition`、画像岗位、已选简历 ID、真实简历列表和难度草稿计算准备状态。未选择简历明确显示“暂不使用”，不将其描述为缺失或错误。

### 保留设置控件和数据属性

继续使用 `interviewPosition`、`interviewResume`、`interviewDifficulty`、`data-interview-action="start"` 和 `data-link="interview-history"`。仅增加包装结构、说明和专属类，不改变开始请求和草稿同步逻辑。

### 进行中与报告复用真实状态

`renderInterviewRoom`、`renderInterviewReport`、`renderInterviewAnalysisPending` 和 `renderInterviewAnalysisUnavailable` 继续读取原状态和报告字段。新增的工作台包装不得触发额外重绘、自动提交、自动结束或虚假评分。

### 窄屏统一单列

桌面端准备概览和设置可以双列展示，窄屏切换为单列；所有输入、问题、回答、报告卡和按钮允许收缩换行，并保留清晰焦点。

## Risks / Trade-offs

- [结构重构破坏开始事件] → 保留原 ID 与数据属性，并增加静态契约校验。
- [面试过程重绘影响输入] → 不改变状态更新与事件同步逻辑，只调整初次渲染结构和样式。
- [准备状态被误解为真实结果] → 总览只展示岗位、简历与难度，不展示推测分数或通过状态。
- [长问题和报告撑开页面] → 所有网格子项设置 `min-width: 0` 并允许长文本换行。

## Migration Plan

1. 增加页面模式和真实准备总览。
2. 重构准备设置、局部提示和开始操作。
3. 为进行中、分析状态和报告增加统一工作台样式。
4. 更新静态契约和资源版本，完成视觉及构建验证。

回滚仅涉及前端结构、样式和资源版本，不涉及数据迁移。
