# 变更提案：管理后台题库管理分型

## 背景

当前管理后台的“题库审核”只展示空状态或单一面试题审核入口，用户无法区分“面试题库”和“职业测评题库”。这会让管理端能力看起来不完整，也不符合 CyanCruise 的业务结构：面试练习需要面试题，职业测评需要测评量表和测评题。

## 目标

- 将管理后台菜单从“题库审核”升级为“题库管理”。
- 在题库管理中提供两个分区：面试题库、职业测评题库。
- 面试题库支持新增、编辑、发布、隐藏、删除。
- 职业测评题库接入现有测评量表和题目，并能在管理后台新增、编辑和删除题目。
- 操作提示使用普通用户能理解的中文，不暴露接口路径或 Java 类名。

## 非目标

- 本次不把职业测评题库迁移为平台业务对象持久化存储；新增和删除先作用于应用运行期题库。
- 本次不改变用户端职业测评答题、计分和记录查询逻辑。
- 本次不迁移第三方题库或大模型生成题的完整审核流。

## 影响范围

- `admin-console-governance`：管理后台题库菜单、面试题管理和测评题库查看。
- `assessment-core`：在现有职业测评量表读取能力基础上，补充后台题目保存和删除能力。
- WebAPI 自定义路由：补齐后台面试题保存、更新、删除能力。
