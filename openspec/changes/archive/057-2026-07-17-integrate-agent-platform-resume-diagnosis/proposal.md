## Why

CyanCruise 的简历诊断页面已经能够展示总分、分项评分和结构化修改建议，但当前 AI 调用仍停留在通用聊天网关，尚未接入已搭建的金蝶 Agent 平台简历诊断任务流。需要在不改变页面契约和用户数据归属边界的前提下，让后端受控调用任务流并消费其 JSON 结果。

## What Changes

- 新增金蝶 Agent 平台任务流调用适配器，供服务端以配置化方式调用简历诊断任务流。
- 后端组装简历正文、目标岗位、岗位要求和画像摘要为任务流诊断上下文，禁止浏览器直接调用平台或持有平台凭据。
- 校验任务流返回的诊断 JSON，并复用现有简历诊断解析、保存和页面返回链路。
- 平台调用不可用、超时或返回非法 JSON 时执行有限重试；重试仍失败则向用户返回可重试提示，不生成或保存规则版诊断。
- 为平台任务流输入、输出和失败场景补充契约与回归测试。

## Capabilities

### New Capabilities

- `agent-platform-resume-diagnosis-adapter`: 定义 CyanCruise 服务端调用 Agent 平台简历诊断任务流的请求、响应、鉴权、有限重试和失败边界。
- `agent-only-resume-diagnosis`: 定义仅在智能体返回合法结构化结果后生成、保存和展示简历诊断。
- `resume-diagnosis-history`: 定义按用户和简历保存、查询、复用与删除诊断记录的边界。

### Modified Capabilities

- `resume-diagnosis`: 简历诊断使用平台任务流的结构化结果，保持既有成功结果字段、所有权校验和保存行为；智能体失败时不生成伪诊断。

## Impact

- 影响 `code/base/v620-cc001-base-common/` 中的任务流调用公共契约。
- 影响 `code/cloud01/v620-cc001-cloud01-app01/` 中的简历诊断分析器、平台调用适配器和配置读取。
- 影响现有简历诊断测试；不要求改动现有 Web 页面返回结构。
- 需要确认金蝶 Agent 平台正式 API、服务端鉴权方式、任务流编码和输入输出格式；不新增浏览器侧凭据或直接平台调用。
