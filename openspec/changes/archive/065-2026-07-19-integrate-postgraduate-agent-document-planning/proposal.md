## Why

升学中心虽然已有升学路线、每日行动和金蝶智能体调用骨架，但考研路线生成尚未形成“用户画像 + 目标院校 + 用户上传资料 → 智能体 → 持久化路线与每日行动”的完整闭环。用户也无法在升学中心管理自己的考研资料，导致智能体无法基于招生简章、专业目录、个人成绩材料等内容生成可核对的定制规划。

## What Changes

- 将考研规划任务流作为升学中心考研方向的真实智能生成入口，运行时通过金蝶智能体 SDK 调用已发布的考研规划任务流。
- 在后端统一整理当前用户画像、目标院校、当前日期、现有路线进度和已解析的考研资料，并作为任务流输入；不信任浏览器提交的用户身份或画像快照。
- 为升学中心增加用户资料上传、正文提取、列表和删除能力，资料按当前用户与考研方向隔离。
- 在 PostgreSQL 中持久化资料元数据、文件引用、解析正文和解析状态；路线、阶段、每日行动继续使用独立升学规划表持久化。
- 校验智能体结构化输出；智能体失败或输出无效时保留原路线，不以规则结果冒充智能生成结果。
- 在升学中心增加“规划依据资料”区域，使用户能上传、查看解析状态和删除资料，并明确资料会用于下一次智能路线生成。
- 保持已开始或已完成阶段及其进度，只刷新尚未开始的阶段；每日行动继续从当前有效阶段逐日推进。

## Capabilities

### New Capabilities

- `postgraduate-agent-route-planning`: 定义考研智能体输入、输出校验、路线及每日行动生成、失败保护和运行配置要求。

### Modified Capabilities

- `further-study-companion-storage`: 增加用户考研规划资料的文件引用、解析正文、状态、查询、删除和 PostgreSQL 持久化要求。
- `postgraduate-exam-companion`: 将用户画像、目标院校和用户资料接入考研路线生成，并保证生成结果属于当前用户。
- `webapp-cyancruise-pages`: 在升学中心增加考研资料上传、状态展示和删除交互。

## Impact

- 后端：`StudyPlanApplicationService`、考研智能体适配器、升学中心 WebAPI、自定义统一路由插件、升学中心存储接口及 PostgreSQL/内存实现。
- 公共契约：新增考研规划资料 DTO 和保存/删除请求。
- 数据库：新增升学规划资料表及用户、方向索引；复用现有文件服务保存实际文件。
- 前端：升学中心 API 路由、状态、上传/列表/删除交互与样式。
- 配置：继续使用 `cc001.agent.platform.study.postgraduate.*` 运行时配置，不在业务代码中硬编码令牌、任务流编码或本地路径。
- 兼容性：保持 JDK 1.8、Cosmic WebAPI 和现有就业规划边界不变。
