## Context

测评结果已通过 `AssessmentResultStorage` 持久化，并在完成时写入 `UserProfileSnapshot.AssessmentBlock`。职业画像快照已由 PostgreSQL 存储，因此适合作为最新深度画像的持久化位置。现有简历诊断通过服务端 Agent SDK 调用任务流，前端只访问 CyanCruise WebAPI。

## Goals / Non-Goals

**Goals:**

- 在完成至少一项测评后，由用户主动触发真实的 Agent 平台 AI 分析。
- 仅向 AI 传递已完成测评结果和已保存的用户事实画像；校验返回 JSON 后持久化最新画像。
- 明确展示证据、可信度和资料不足，供用户查看并供下游能力读取。
- AI 不可用或返回无效 JSON 时返回可重试错误，不生成规则兜底画像。

**Non-Goals:**

- 不覆盖用户自行填写的教育、目标岗位、经历等事实字段。
- 不提供跨用户画像比较、历史版本编辑或历史版本恢复为最新画像的能力。
- 不在浏览器中暴露 Agent 编码、凭据或平台调用细节。

## Decisions

### 使用快照中的独立 AI 深度画像区块

新增 `UserProfileSnapshot.AiDeepProfileBlock`，将摘要、标签、偏好、发展建议、证据、资料不足、生成时间和来源作为独立字段保存。相比新增单独表，这能复用已运行的 PostgreSQL 快照持久化、下游快照读取和用户隔离能力；相比将内容混入 `AssessmentBlock`，它能明确区分“测评分数”与“AI 推断”。

### 自画像补充作为用户事实保存

在 `OnboardingBlock` 中新增 `selfProfileSupplement`，用于保存用户针对深度画像资料不足项主动补充的项目职责、技术基础、学习约束、协作经历或目标限制等真实信息。该字段属于用户事实，不与 AI 推断合并；深度画像、简历诊断和后续规划组装上下文时优先读取它。再次生成深度画像时，模型不得把该字段已经明确提供的内容重复列为资料不足。

### 由后端汇总输入并调用 Agent SDK

新增深度画像应用服务和分析器。它从 `AssessmentResultStorage` 读取用户最新的已完成测评，并从职业画像快照读取用户事实，通过现有 Kingdee Agent SDK 客户端发送单一 JSON `question`。这与简历诊断的受控调用模式一致，避免浏览器持有平台凭据。

### 单项测评解读与最终深度画像分两层生成

测评完成后，后端保留题目文本、所选项文本、维度和得分等答题快照。用户可主动为单份测评生成 AI 解读，任务流以 `ASSESSMENT_INTERPRETATION` 模式返回该测评的观察、建议和资料不足；用户再主动生成深度画像时，任务流以 `DEEP_PROFILE` 模式汇总全部已完成测评、已有单项 AI 解读和用户事实画像。两类结果均只使用真实输入，调用失败时仅提示重试，不以规则结果替代。

### 采用严格 JSON 校验和无兜底失败策略

分析器仅接受包含 `profileSummary`、标签、偏好、证据和资料不足项的 JSON 对象。响应无效、平台未配置或调用失败时抛出可读错误；页面保留上一次已保存画像，不显示伪造的规则分析。

### 下游仅读取最新有效深度画像

快照合并后，简历诊断、就业规划和升学规划的上下文组装会附加 `aiDeepProfile` 摘要和标签，并标记为“基于测评的 AI 分析”。下游仍优先使用用户明确填写的事实字段。

### 最新画像与历史记录共同保存在职业画像快照

`aiDeepProfile` 继续保存当前最新有效画像，供简历诊断、就业规划和升学规划稳定读取；`aiDeepProfileHistory` 按生成时间倒序保存历次成功结果，并为每条记录分配 `recordId`。为控制快照体积，仅保留最近 20 条历史记录。画像补全页只展示最新摘要，独立详情页负责展示完整画像和历史版本。

## Risks / Trade-offs

- [尚未配置画像补全智能体编码] → 后端返回“深度画像暂未配置，请稍后重试”，不会写入空结果。
- [模型对测评推断过度] → 系统提示词要求证据和可信度，页面提示其为参考分析；无证据的结论写入资料不足而非能力判断。
- [多次生成结果不同] → 每次成功生成覆盖为最新深度画像，页面显示生成时间和来源；下游只读取这条最新有效记录。
- [测评结果不足] → 后端拒绝调用并提示至少完成一项测评。

## Migration Plan

1. 部署 DTO、快照合并、应用服务、WebAPI 和页面变更。
2. 在运行环境配置 `cc001.agent.platform.profile.enabled=true` 与画像补全智能体编码。
3. 重启 `MC_5`，完成测评后生成并查看深度画像。
4. 配置错误时关闭该开关即可停止新调用，已保存的用户事实和旧快照不会受影响。
