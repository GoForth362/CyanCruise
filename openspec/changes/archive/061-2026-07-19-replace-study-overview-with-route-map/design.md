## Context

`further-study-home` 当前由通用 `renderFeatureHome` 渲染，只显示考研、保研、留学三张功能卡。就业页已经采用“路线图在前、工具和资源在后”的专用渲染方式，用户希望升学路线获得相同的信息层级。现有自画像已经提供路线选择、教育阶段、学校和专业，可在不增加后端接口的前提下生成第一版规则型升学路线图。

## Goals / Non-Goals

**Goals:**

- 将 `further-study-home` 从工具总览替换为升学路线图页面。
- 首页选择深造后，路线入口稳定进入 `further-study-home`。
- 路线图展示当前准备情况、待补信息和四个升学阶段，并结合方向选择、洞察和资讯形成完整升学中心。
- 删除页面底部重复的考研、保研、留学陪伴卡片，但保持现有三个陪伴页面和接口不变。

**Non-Goals:**

- 不新增升学规划智能体、数据库表或独立持久化流程。
- 不自动替用户决定考研、保研或留学方向。
- 不重写考研、保研、留学陪伴页面内部表单。

## Decisions

1. 复用 `further-study-home` route，不新增第二个升学主页 route。这样首页入口、平台菜单和已有深造子页面父级无需迁移 URL，只替换页面语义和渲染器。
2. 新增专用 `renderFurtherStudyHome`，复用就业路线图的 `roadmap-section`、`roadmap-panel`、`roadmap-summary` 和 `roadmap-step` 视觉结构，避免维护两套布局。
3. 第一版路线图采用前端规则生成。它读取 `snapshot.onboarding` 和 `snapshot.preferences` 中的教育阶段、学校、专业及路线信息，仅展示已有事实和明确的待补提示，不宣称已经生成智能规划。
4. 自画像目标字段按路线拆分：就业使用 `targetRole`，深造使用 `targetSchool`。前端切换路线时保留两类输入的独立草稿，后端沿用现有 onboarding 保存接口，在请求和快照中增加可选 `targetSchool`，不新增接口或数据表。
5. 四个固定阶段为“明确升学方向”“评估基础与差距”“制定备考或申请计划”“推进材料与关键节点”。具体方向通过升学中心顶部选择器维护，底部不再重复显示三张陪伴卡片。
6. 平台挂载名称从“深造护航”统一为“升学路线图”，但 route key 保持 `further-study-home`，降低部署和回滚风险。

## Risks / Trade-offs

- [规则路线图不包含智能体个性化结果] → 明确标注“根据当前自画像整理”，并把缺失信息展示为待补项，不使用“智能生成”文案。
- [用户尚未选择具体升学方向] → 路线图第一阶段显示“待选择”，并使用顶部方向选择器完成选择。
- [旧平台菜单仍缓存旧标题] → 更新平台挂载配置和前端缓存版本；部署后重新发布或刷新对应菜单配置。
- [复用就业路线样式导致语义混淆] → 仅复用布局类，所有用户文案改为“升学”“备考”“申请”“材料”等表达。

## Migration Plan

1. 部署更新后的静态资源、route metadata 和平台挂载说明。
2. 保持 `ccRoute=further-study-home` 不变，旧链接自动展示新路线图。
3. 验证首页深造入口、平台菜单、顶部方向选择及升学资讯展示。
4. 如需回滚，只恢复 `further-study-home` 的通用功能卡渲染和旧标题，不涉及数据回滚。

## Open Questions

无。后续如接入升学规划智能体，应另建 change 定义后端契约和持久化边界。
