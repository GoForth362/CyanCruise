## Why

当前用户管理列表展示并搜索用户学校、专业，还直接展示平台组织 ID。学校和专业属于用户个人画像数据，不应进入账号治理视图；平台组织 ID 仅用于服务端组织归属和权限隔离，在单组织用户管理页面没有直接操作价值。

## What Changes

- 用户管理接口只返回账号治理所需信息，不再返回学校、专业和平台组织 ID。
- 用户管理不再从职业画像复制学校和专业，也不再按这些字段搜索。
- 用户管理页面移除学校/专业与平台组织 ID 列，搜索范围收敛为姓名和用户 ID。
- 平台组织 ID 继续保留在服务端存储和组织权限逻辑中，不改变组织隔离行为。

## Capabilities

### New Capabilities

无。

### Modified Capabilities

- `admin-console-governance`: 用户管理视图和接口遵循个人数据最小化原则，仅暴露账号治理所需字段。

## Impact

- 影响管理员用户列表与用户详情 WebAPI 返回契约。
- 影响 `AdminConsoleGovernanceApplicationService` 的列表、详情、搜索和画像同步逻辑。
- 影响 `webapp` 管理后台用户列表与搜索文案。
- 不修改用户职业画像存储，也不改变平台组织权限和数据隔离。
