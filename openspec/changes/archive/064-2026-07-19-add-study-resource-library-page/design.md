## Context

升学中心已经通过 `/cc001/study-center/resources/list` 读取独立维护的升学内容，并在 `further-study-home` 按“官方服务、精选文章、相关视频”三列展示。当前页面直接展示接口返回的全部可见内容，默认数据数量较少，且没有对应就业中心 `career-resources` 的独立资源库 route。

本变更跨越升学资源默认数据、Web 路由元数据、页面渲染与静态契约验证。实现必须兼容 JDK 8，继续使用既有接口，不把升学内容混入就业资源存储。

## Goals / Non-Goals

**Goals:**

- 扩充真实、清晰、面向学生的默认升学资源集合。
- 让升学中心首页只承担摘要展示，并提供明显的“全部资源”入口。
- 新增可直接访问、可返回、具备加载/错误/空状态的全部升学资源页面。
- 用路由验证脚本覆盖新 route、按钮和 API 依赖，防止后续回归。

**Non-Goals:**

- 不新增搜索、分页、收藏或个性化推荐接口。
- 不抓取或复制外部文章正文，只维护标题、摘要、分类与外部链接。
- 不调整管理后台现有升学资讯维护接口。
- 不修改就业资源页及其数据语义。

## Decisions

### 1. 复用现有升学资源列表接口

首页与全部资源页都复用 `studyCenterResources` 对应的 `/cc001/study-center/resources/list`。前端只维护一份加载状态和数据集合，避免引入功能重复的接口。

替代方案是新增 `/all` 或分页接口，但当前默认资源规模较小，不足以证明额外 API 与 DTO 的必要性。

### 2. 首页限制每栏摘要数量，独立页面展示完整集合

首页按类型分组后仅展示每栏前两条，维持三列布局和页面节奏；“全部资源”页面展示所有可见内容。分组顺序继续使用官方服务、精选文章、相关视频，未知类型归入精选文章，避免资源无处展示。

### 3. 新增稳定 route `study-resources`

新页面使用 `study-resources`，其父级返回关系为 `further-study-home`，并在路由清单中声明 `studyCenterResources` API 依赖。页面不加入默认平台侧边栏，主要从升学中心按钮进入，同时保留 URL 直达能力。

### 4. 默认数据由升学中心存储维护

在现有升学资源默认集合中补充考研、保研、留学和官方服务内容，使用普通用户能理解的中文标题与摘要。外部资源只保存公开入口链接，不保存第三方正文。

## Risks / Trade-offs

- [外部链接可能变更] → 优先选择长期稳定的官方站点入口，并通过管理后台允许后续维护。
- [首页资源增多导致页面过长] → 首页每个分组限制展示两条，完整集合移至独立页面。
- [部分资源类型值不规范] → 沿用前端既有分类兜底逻辑，将未知类型归入文章区。
- [接口不可用时新页面无内容] → 提供独立加载、错误和空状态，并保留返回升学中心操作。

## Migration Plan

1. 部署包含扩充默认资源的业务 JAR。
2. 部署新增 route 和页面渲染逻辑的 Web 资源。
3. 运行路由校验与相关 Java 测试。
4. 若需回滚，可恢复旧 Web 资源；新增默认资源不改变数据结构，也不影响已有记录。

## Open Questions

无。搜索、筛选和分页将在资源规模显著增长后另行设计。
