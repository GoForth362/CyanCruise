## Context

“画像补全”首页由 `app-runtime.js` 直接拼接页面结构，并复用 `styles.css` 中的通用卡片样式。现有布局把深度画像入口和题组列表分别放在宽幅白色区块中，缺少进度概览与明确的下一步提示；题组使用五列横向结构，在中小屏幕上容易拥挤。现有数据已经能够提供题组总数、实际题量、已完成题组、最新深度画像和生成时间，无需新增接口。

## Goals / Non-Goals

**Goals:**

- 用真实数据呈现画像完成进度和最新深度画像状态。
- 把题组改为响应式卡片网格，并统一使用不同明度的青色色阶。
- 保持所有现有操作的数据属性和事件绑定不变。
- 让首页在窄屏、键盘操作和减少动态效果偏好下仍然可用。

**Non-Goals:**

- 不调整测评题目、计分、结果、深度画像生成或历史记录接口。
- 不改变题组顺序、完成状态判定和“查看上次结果”的业务逻辑。
- 不重构进入题组后的作答页和结果详情页。

## Decisions

### 使用专属页面状态类隔离首页样式

仅在画像补全题组首页为 `body` 添加 `assessment-dashboard-mode`，进入作答或结果页后自动移除。这样可以强化首页页头与容器视觉，同时避免新样式影响现有作答流程。相比覆盖通用 `.assessment-card`，专属类的回归范围更小。

### 使用现有状态计算真实进度

首页继续以 `completedAssessmentScaleIds()` 判定已完成题组，以加载到的题组数组作为总数，展示“已完成 / 总题组”。不引入演示数字，也不把深度画像生成状态误当作题组完成状态。

### 将题组横向行改为双列卡片

桌面端采用双列网格，卡片内部按“编号与状态—标题说明—元信息—操作”垂直组织；窄屏降为单列。各卡片使用同一青色系的不同明度，不再使用与整体风格不一致的粉色图标。

### 保留事件数据属性

继续使用 `data-assessment-scale`、`data-deep-profile-generate` 和 `data-link="deep-profile-detail"`，避免改变现有事件委托与接口调用。视觉重构只调整 HTML 包装层级和 CSS 类名。

## Risks / Trade-offs

- [通用页面样式误伤作答页] → 通过 `assessment-dashboard-mode` 和 `assessment-dashboard-*` 专属选择器隔离。
- [已完成数量大于当前可见题组数] → 展示进度时将已完成数量限制在当前题组总数内，题组自身仍按原映射判定。
- [卡片说明长度不一致导致布局跳动] → 桌面端统一卡片结构与最小高度，窄屏取消不必要的固定高度。
- [动画造成页面闪动或不适] → 仅使用轻量悬浮过渡，并在 `prefers-reduced-motion` 下禁用。

## Migration Plan

1. 更新首页渲染结构和页面状态类。
2. 增加专属响应式样式与可访问性状态。
3. 扩展静态契约校验并更新资源版本。
4. 运行 OpenSpec、前端静态校验和项目构建。

回滚时可独立撤销本变更涉及的首页结构、专属样式和资源版本，不涉及数据迁移。
