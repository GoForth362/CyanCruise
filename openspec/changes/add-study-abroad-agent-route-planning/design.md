## Context

升学中心的规划表、每日任务表和资料表已经包含方向字段，并且现有应用服务能够按 `POSTGRADUATE`、`RECOMMENDATION`、`STUDY_ABROAD` 选择独立智能配置。当前缺口主要是留学资料被前后端白名单排除、生成输入缺少留学语义约束、完整路线校验未覆盖留学，以及页面按钮和空状态仍使用笼统升学文案。

本次变更跨越智能生成器、资料服务、页面展示和测试，但不改变共享 JSON 契约、不新增依赖或数据库表，必须兼容 JDK 1.8 和 Cosmic 运行时。

## Goals / Non-Goals

**Goals:**

- 留学复用考研、保研的真实智能规划调用、共享解析、阶段进度保护和今日行动持久化。
- 留学资料、规划和任务严格按用户与 `STUDY_ABROAD` 方向隔离。
- 预留 `cc001.agent.platform.study.abroad.*` 配置，支持任务流编码优先、智能体编号作为可选入口。
- 未配置、超时或格式无效时保持真实空状态或原有有效路线，不写入兜底数据。
- 页面用普通中文准确展示留学规划、资料和今日行动状态。

**Non-Goals:**

- 不在代码中内置院校、项目、语言考试、申请截止时间或签证规则。
- 不替用户自动选择国家、学校、专业或申请批次。
- 不改造现有留学陪伴的单项诊断接口。
- 不硬编码智能体编号、任务流编码或认证信息。

## Decisions

### 1. 复用方向化共享存储，不新建留学专用表

规划和今日任务继续使用 `(user_id, direction)` 与 `(user_id, direction, task_id)` 复合键，规划资料继续按用户与方向查询。这样可以直接获得跨实例持久化和三方向隔离，避免平行表产生重复逻辑。

### 2. 留学使用独立配置并采用任务流优先策略

`AgentPlatformStudyPlanGenerator` 继续读取 `cc001.agent.platform.study.abroad.enabled`、`agentNumber` 和 `taskFlowCode`。配置任务流编码时由共享 SDK 客户端优先直连任务流，行为与考研、保研一致；未配置任何可用入口时生成失败且不落库。

### 3. 共享结构不变，增加留学语义和完整性校验

留学仍输出共享的 `targetRole`、`startStateSummary`、`phases`、`weeklyPlan`、`dailySuggestions`、`weeklyFocus`。生成输入明确要求围绕目标国家或地区、院校与专业定位、语言或标准化考试、背景提升、选校、文书、网申、录取后事项安排，并将未核验信息标记为待确认。路线必须覆盖未来十二个月且包含至少三个阶段，不能用当前一个月的默认阶段通过校验。

### 4. 资料只进入当前用户的留学生成请求

前后端资料白名单扩展到 `STUDY_ABROAD`，文件目录继续包含用户和方向。只有正文提取成功的当前用户留学资料会注入生成请求，考研、保研资料不会交叉读取，也不会写入公共知识库。

### 5. 空状态与生成失败保持真实

`ensure` 只读取已持久化且校验通过的路线；显式“生成留学规划”才调用智能服务。无路线时完整路径和今日行动显示留学专属空状态，绝不创建示例阶段、规则任务或浏览器本地预览路线。

## Risks / Trade-offs

- [留学智能体仍返回考研语义] → 输入固定 `STUDY_ABROAD` 并增加留学目标与结构校验，失败结果不保存。
- [不同国家申请周期差异较大] → 允许缺失信息进入第一阶段核验任务，不在代码中假定固定申请季。
- [任务流尚未配置] → 页面返回清晰可恢复错误并保留原有有效路线。
- [上传资料跨方向串用] → 文件目录、查询条件和删除校验同时约束用户与方向。

## Migration Plan

1. 部署留学方向后端白名单、语义输入与完整路线校验。
2. 部署升学中心、完整路径和今日行动的留学文案及资料入口。
3. 在未配置智能服务时验证无兜底数据，再配置留学智能体或任务流完成真实生成验收。
4. 回滚仅需撤回本次代码和页面资源；既有方向化数据结构无需回滚。
