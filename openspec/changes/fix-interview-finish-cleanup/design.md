## Context

普通 AI 模拟面试和全景仿真面试均在 `app-runtime.js` 中以页面状态和 Promise 回调推进。全景模式还持有 `MediaStream`、语音识别对象、语音合成和倒计时器。当前主动结束只先停止部分能力，完整媒体清理位于复盘请求成功回调中；问答请求也没有独立的会话代次校验。

## Goals / Non-Goals

**Goals:**

- 点击结束的同步阶段立即停止本地音视频、语音能力和计时。
- 让结束动作成为不可逆的当前页面状态，任何较晚返回的问答回调都不能继续更新题目。
- 复盘接口成功或失败都不恢复设备和问答；成功后仍可查看复盘。
- 普通文字面试和全景面试使用同样的异步失效原则。
- 零回答退出时持久化结束会话并直接返回准备主界面，不触发 AI 复盘。

**Non-Goals:**

- 不取消已经发往服务器的 HTTP 请求，因为现有请求封装未暴露 `AbortController`。
- 不修改复盘生成逻辑或持久化模型。
- 不改变自动完成第七题后生成复盘的业务规则。

## Decisions

1. 为两类面试分别维护递增的“会话操作令牌”。开始新面试、主动结束或重置时更新令牌；异步问答和结束回调仅在令牌仍匹配时更新当前页面。相比只检查 `busy`，令牌可以识别已经结束或已被新会话替代的旧请求。
2. 全景面试主动结束时先调用统一媒体清理，再发起复盘请求。清理函数停止语音识别、倒计时、题目朗读和 `MediaStream` 的全部轨道，因此摄像头和麦克风不会依赖网络结果释放。
3. 点击结束后立即清空当前题目并设置退出状态，禁止回答、听写和朗读入口继续工作。复盘成功后展示报告；失败后保持已退出状态并显示“复盘生成失败，可重试”，但不重新进入面试房间。
4. 普通文字面试结束时同步停止语音输入，并让尚未完成的回答回调失效。自动完成第七题属于同一有效操作链，不在其内部提前使自己的结束请求失效。
5. 路由离开和浏览器卸载仍保留现有兜底清理，主动结束新增同步清理作为首要保障。
6. 新增“仅结束会话”WebAPI，内部复用 `InterviewApplicationService.end`。两类页面在回答数为 0 时调用该接口，成功或失败均清空本地会话并返回各自准备主界面；有回答时继续调用“结束并生成复盘”。相比直接丢弃本地状态，该设计不会让数据库中的零回答记录长期保持“进行中”。

## Risks / Trade-offs

- [结束请求仍在后台执行] → 页面只忽略过期回调，不尝试中断后端持久化；这是为了兼容现有请求封装，同时保证用户端不再继续问答。
- [复盘失败后不能恢复原面试] → 明确提供重试复盘或查看记录的入口，避免为了恢复交互而重新占用媒体设备。
- [令牌遗漏导致旧回调更新状态] → 在静态契约校验中检查令牌创建、主动结束递增和问答回调守卫。
- [仅结束接口调用失败] → 仍优先完成浏览器资源释放并返回主界面，同时给出退出提示；后端异常不允许重新占用设备。
