## Context

三个深造陪伴首页已经拆分出独立子路由，但首页仍沿用早期占位式结构：先展示一排阶段标签，再以几乎相同的大按钮重复展示一次。它既没有提供更多决策信息，也没有形成陪伴产品应有的阶段感。当前三个首页使用相同的 `postgraduate-hero-panel`、`postgraduate-flow` 和 `postgraduate-action-grid`，适合提取成统一的行动首页渲染器。

## Goals / Non-Goals

**Goals:**

- 用一次点击入口同时表达阶段、任务、功能价值和下一步。
- 三个页面保持统一操作模型，同时通过主题、文案和阶段数量形成差异。
- 保留现有 `data-link` 路由机制、键盘操作和子页面。
- 在桌面、平板和手机宽度下保持清晰的阅读顺序。

**Non-Goals:**

- 不重做子页面表单和结果页面。
- 不引入图片素材、图标库或前端框架。
- 不新增进度数据、后端接口或虚构用户状态。

## Decisions

1. 新增 `renderCompanionHub(config)` 统一生成首页，配置包含主题、定位文案、阶段卡片和辅助标签，避免三个页面继续复制结构。
2. 首页由“主题引导区 + 阶段轨道 + 行动卡片”组成。阶段轨道用于建立完整路径认知，行动卡片负责实际跳转，不再展示第二组重复按钮。
3. 卡片使用原生 `<button data-link>`，包含阶段编号、行动短语、功能名、说明和“进入功能”反馈，兼容现有事件代理与键盘交互。
4. 三个主题统一使用青色体系，只通过 CSS 自定义属性和修饰类调整明度与冷暖倾向：考研使用明亮青绿，保研使用沉稳青色，留学使用偏冷蓝青。这样既保持“深造护航”的产品一致性，又能在不引入割裂色彩的情况下识别不同路径。
5. 桌面端根据卡片数量自适应网格，窄屏切为单列；悬停和键盘焦点提供抬升、边框及箭头反馈，并尊重减少动态效果的系统设置。

## Risks / Trade-offs

- [视觉增强影响宿主一致性] → 保留现有字体、按钮事件和主色体系，主题色仅用于局部层次。
- [卡片说明增加页面高度] → 删除重复入口抵消高度，文案限制为一行任务和简短说明。
- [通用渲染器掩盖路径差异] → 每条路径独立配置任务语言、说明、主题和阶段数量。
- [旧样式残留造成冲突] → 新结构使用独立 `companion-hub-*` 类，并在校验中禁止三个首页继续使用旧重复网格。
