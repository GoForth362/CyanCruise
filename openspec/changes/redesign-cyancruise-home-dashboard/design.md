## Context

生产首页由 `renderHomeIntentPage` 读取当前用户画像、路线、简历、面试和规划状态，并通过通用页面壳展示。当前已保存自画像占据较大的整行面板，`overviewStrip` 将目标、方向、今日行动和路径规划按同等尺寸展示；当今日行动返回较长任务文本时，状态卡会显著增高，路线入口被推到首屏下方。此次调整只重排这些已有数据和操作，不改变加载、保存或路由逻辑。

## Goals / Non-Goals

**Goals:**

- 首屏优先表达当前路线、目标和今天最值得推进的行动。
- 保留自画像展开、修改、当前方向中心、路线规划等既有操作。
- 让目标、方向、简历、面试和规划状态保持可扫描且高度稳定。
- 使用与现有产品一致的青色视觉，并适配桌面、平板和窄屏。
- 所有结论只来自已经加载的用户数据，缺失时展示明确等待状态。

**Non-Goals:**

- 不修改画像、今日行动、简历、面试或规划接口。
- 不新增进度百分比、连续天数、完成趋势等后端没有提供的数据。
- 不改变就业与升学路线的数据隔离、保存和切换规则。
- 不重做自画像编辑表单或各功能子页面。
- 不引入图片、图标库、前端框架或第三方依赖。

## Decisions

1. 为非调试工作台增加 `home-dashboard-mode` 页面状态，只定向作用于首页页面壳和新样式，避免影响其他功能页。
2. 已保存状态使用“路线欢迎区 + 紧凑自画像抽屉”；编辑状态继续使用原表单，保证表单字段和保存逻辑不变。
3. 新增 `homeDashboardOverview(goal)`，将 `overviewRows(goal)` 的真实数据拆为一张今日重点卡和三张稳定高度的状态卡。今日行动完整文本保留在 DOM 和 `title` 中，视觉上限制行数，并提供进入今日行动页的明确按钮。
4. 新增首页专属入口卡片渲染器，仍使用现有 `data-link`、`data-platform-link` 和页面事件代理，不新增导航实现。
5. 首页主题统一使用不同层次的青色、白色和少量中性色；悬停只提供轻微抬升，键盘焦点使用清晰轮廓，并尊重减少动态效果设置。
6. 宽屏使用摘要双栏、状态三列和入口四列；中等宽度逐级降列，窄屏保持单列阅读顺序。

## Risks / Trade-offs

- [今日行动视觉截断] → DOM 与 `title` 保留完整文字，并提供“查看今日行动”入口。
- [首页专属样式影响其他页面] → 所有新增规则限定在 `.home-dashboard-mode` 或 `.home-dashboard-*` 类下。
- [紧凑自画像降低部分信息可见性] → 默认保留路线、目标、身份和简历状态，完整信息仍可通过原“展开自画像”查看。
- [路线入口与今日重点存在部分重复] → 今日重点表达“现在做什么”，入口区表达“还能去哪里”，信息目的不同。

## Migration Plan

1. 发布新的首页渲染、样式和静态资源版本。
2. 分别验证就业、深造、无规划、长今日行动和自画像编辑状态。
3. 如需回退，仅恢复前端渲染和样式文件；用户数据与后端无需迁移。
