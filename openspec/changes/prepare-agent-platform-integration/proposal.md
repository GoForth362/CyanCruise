## Why

CyanCruise 需要形成覆盖职业测评、用户画像、就业、考研、保研、留学、简历和模拟面试的完整学生成长闭环。当前平台已建立部分智能体和三个知识库，但尚未形成可按用户意图稳定运行的任务流组合。需要先明确平台侧可独立验收的搭建范围，并为后续后端接入预留清晰边界，避免把平台演示、业务数据写入和应用集成混在一起。

## What Changes

- 定义通用应用助手、生涯发展主调度智能体、用户画像、人格分析、信息检索、就业规划、简历优化、模拟面试、考研规划、保研规划和留学规划的职责、绑定关系及验收场景。
- 建立生涯发展主调度流程：画像识别、人格结果参考、事实检索、专业任务流调用和结果汇总。
- 定义考研、保研、留学、就业、简历优化、就业面试、升学面试和职业测评分析等可独立测试的任务流；任务流输出均使用普通用户可理解的中文。
- 定义平台发布前的测试集、知识库授权检查和失败提示要求。
- 提供就业规划任务流的节点、变量、提示词和结构化输出蓝图，并通过官方 Agent SDK 从 CyanCruise 服务端调用已发布的就业规划智能体。
- 将用户主动生成的结构化就业计划保存到既有职业计划边界，并由现有今日行动页面读取计划中的本周动作和每日建议；浏览器不持有平台凭据。
