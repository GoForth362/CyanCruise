## Context

管理后台已具备独立导航、账号区域、总览、用户、内容、题库、公告和审计板块，接口与治理动作也已稳定。当前主要问题集中在总览的信息表达：四张指标卡、四块状态面板和最近操作使用近似的视觉重量，管理员难以快速区分平台规模、健康程度和需要处理的事项。此次调整限定在现有管理前端模块和样式层，不改变任何治理数据或授权边界。

## Goals / Non-Goals

**Goals:**

- 让管理员首先看懂“平台现在怎么样”和“接下来需要处理什么”。
- 以统一青色体系建立专业、克制且与 CyanCruise 用户端一致的管理视觉。
- 保留现有真实数据字段，并通过状态色、进度表达和行动入口增强可扫描性。
- 优化桌面、平板和窄屏下的导航与内容阅读顺序。
- 继续支持键盘操作、清晰焦点和减少动态效果设置。

**Non-Goals:**

- 不修改管理接口、数据模型、授权、审计或治理业务规则。
- 不新增趋势、同比、在线率等当前接口没有提供的虚构指标。
- 不重做用户管理、内容管理、题库管理等业务表单的功能流程。
- 不引入图表库、图标库、前端框架或图片素材。

## Decisions

1. 总览使用“欢迎摘要区 + 核心指标卡 + 双栏治理面板 + 最近操作”的阅读顺序。摘要区只使用现有用户、面试、内容和告警数据生成普通中文结论。
2. 核心指标卡增加短编号、主题层次和状态提示，但数字仍直接来自现有 `dashboard`、`analytics` 和已加载集合，不计算无法验证的业务趋势。
3. 将运营、内容和题库状态整合为更紧凑的健康度面板，待关注事项单独提高优先级；没有告警时明确展示正常状态，不制造问题。
4. 常用治理入口复用现有标签切换机制，通过 `data-admin-jump` 切换到用户、内容、题库或公告板块，不增加新路由和接口。
5. 后台样式使用独立 `admin-dashboard-*` 类，统一青色主轴并保留少量橙色警示与红色危险状态；不影响用户端页面。
6. 宽屏使用分栏布局，中等宽度降低列数，窄屏将内侧导航改为横向可滚动标签并保持内容单列。
7. 内容、面试题和职业测评题删除统一复用全局 `showConfirmDialog` 危险操作样式；管理页面模块通过渲染上下文调用该组件，不使用 `window.confirm`，也不改变原删除服务调用。

## Risks / Trade-offs

- [摘要区增加页面高度] → 压缩重复面板标题和指标卡高度，使首屏仍能看到主要健康状态。
- [状态颜色可能被误解为业务结论] → 仅对已有禁用、隐藏、跳过或缺失数据使用警示色，并同时展示文字和数值。
- [新增快速入口与现有导航重复] → 快速入口只作为待办上下文中的捷径，仍调用同一标签切换逻辑。
- [旧样式与新结构冲突] → 新总览使用独立类名，静态校验要求关键结构存在并禁止恢复旧总览函数。

## Migration Plan

1. 部署新的管理页面模块、样式和静态资源版本。
2. 使用现有管理员身份验证总览和六个管理板块切换。
3. 若出现视觉兼容问题，可回退页面模块和样式文件；接口与数据不需要迁移或回滚。
