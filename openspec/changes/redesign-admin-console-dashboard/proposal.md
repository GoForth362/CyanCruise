## Why

管理后台总览目前以尺寸接近的白色报表卡片连续堆叠，关键状态、异常事项和常用管理入口缺少优先级，管理员需要逐块扫描才能判断平台是否正常。需要在不改变数据、权限和治理操作的前提下，重构为更清晰、更高效的管理工作台。

## What Changes

- 将总览重构为“管理摘要 + 核心指标 + 运营健康度 + 待处理事项”的信息层级。
- 使用统一青色体系、轻量状态色和更紧凑的指标卡，增强正常、关注和异常状态的辨识度。
- 优化后台内侧导航、顶部账号区域、内容面板和表格区域的视觉一致性。
- 为常用管理板块提供清晰入口，并补充键盘焦点、窄屏布局和减少动态效果支持。
- 保留现有管理菜单、数据字段、接口调用、权限校验和所有治理操作，不新增虚构指标。

## Capabilities

### New Capabilities

无。

### Modified Capabilities

- `admin-console-governance`: 管理后台总览需要以有优先级的管理工作台呈现现有指标、健康状态、待关注事项和治理入口，并保持现有权限与数据真实性边界。

## Impact

- 修改 `webapp/isv/v620/cyancruise/assets/pages/admin-console.js` 的总览结构。
- 修改 `webapp/isv/v620/cyancruise/assets/styles.css` 的管理后台视觉、状态反馈和响应式样式。
- 更新管理后台静态契约校验与前端资源版本。
- 不修改 `/cc001/admin/*` 接口、后端服务、数据模型或权限规则，不新增第三方依赖。
