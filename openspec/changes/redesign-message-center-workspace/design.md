## Context

消息中心由 `assets/pages/messages.js` 独立页面模块渲染，数据来自通知列表、未读数和订阅配额三个既有接口。页面已经通过内存缓存和动作级按钮状态支持异步刷新，并通过 `message-detail` 路由展示详情。此次优化涉及页面模块、全局页面模式、样式、静态契约和资源版本，但不改变后端数据或接口。

## Goals / Non-Goals

**Goals:**

- 清晰区分当前消息状态、常用操作、通知列表与外部提醒额度。
- 只使用真实通知、未读数和配额生成概览与状态文案。
- 提升管理员公告、未读消息、已读消息、空状态和详情的视觉识别度。
- 保留异步动作级更新，避免刷新、已读或归档操作使整页回到加载状态。
- 适配窄屏、长标题、长正文、键盘操作和减少动态效果偏好。

**Non-Goals:**

- 不修改通知、订阅或周报 WebAPI。
- 不修改消息类型分类、用户归属、分页数量、已读和归档规则。
- 不新增筛选接口、搜索、批量删除或外部推送配置。
- 不虚构通知、待办、活跃度或订阅额度。

## Decisions

### 使用隔离的消息中心页面模式

`messages` 与 `message-detail` 路由启用 `message-center-workspace-mode`，所有新增页面背景、头部和工作台样式均限定在该模式下，避免影响其他普通列表页面。

### 保留页面模块和动作契约

继续使用 `data-message-action`、`data-message-id`、`bindMessages`、`loadMessages`、`applyRead`、`applyAllRead` 与 `applyArchive`。结构重排不改变动作名称和请求参数，操作后仍先更新本地真实缓存，再异步同步服务端数据。

### 概览完全来源于真实数据

未读消息使用 `data.unread`，全部消息使用 `data.notifications.length`，订阅配额使用 `quotaSummary(data.quotas)`。空状态根据真实消息数组判断，不展示模拟趋势、虚假优先级或不存在的待办数量。

### 列表使用状态轨道而非密集按钮卡片

消息项增加类型图标、未读点、来源标签和清晰的操作区，但继续使用原消息标题、正文、类型、时间和动作。管理员公告只基于 `ADMIN_BROADCAST` 判断，未知类型继续按系统消息降级。

### 窄屏改为单列工作流

概览卡片、工具栏、消息项和详情在窄屏依次堆叠；操作按钮可换行或整行显示，长文本自然换行且不产生横向滚动。

## Risks / Trade-offs

- [页面模块文案与结构变化导致契约选择器失效] → 保留所有 `data-message-action`、消息 ID 和路由操作，并扩展静态校验。
- [概览和列表状态短暂不一致] → 继续沿用本地缓存更新函数，操作完成后调用既有刷新同步真实数据。
- [空状态过度强调无消息] → 同时说明消息类型、隐私边界和可用操作，不构造虚假内容。
- [长公告导致页面过高] → 列表正文使用可读截断，完整正文仍在详情页展示。

## Migration Plan

1. 增加消息中心隔离页面模式与真实状态辅助函数。
2. 重构概览、工具栏、列表、空状态和详情结构。
3. 增加青色视觉、响应式、焦点和减少动态效果样式。
4. 更新静态契约和资源版本，完成视觉、OpenSpec、前端和 JDK 8 验证。

回滚只涉及前端结构、样式与资源版本，不涉及数据迁移。
