## Context

全景仿真面试的进行中页面由 `renderPanoramaInterviewPage` 生成。当前语音识别启动或失败、提交回答前、AI 返回下一题后等高频状态变化会调用 `renderPage`，该调用会替换 `pageHost` 的内容，导致 `<video id="panoramaCamera">`、AI 面试官和控制区 DOM 被销毁并重新创建，因此出现画面闪烁。

## Goals / Non-Goals

**Goals:**

- 面试进行中保持全景房间和摄像头 `<video>` 节点引用稳定。
- 使用一个局部同步函数更新题号、状态文案、按钮、计时、回答、错误和提示。
- 进入房间、结束面试、打开报告等阶段切换仍可使用完整渲染。
- 保留现有异步操作令牌，防止旧请求回调更新已结束或已切换的会话。

**Non-Goals:**

- 不改变摄像头、麦克风和语音识别权限申请方式。
- 不修改问题数量、答题计时、后端接口和记录保存规则。
- 不引入前端框架或新增依赖。

## Decisions

1. 为进行中房间增加稳定根节点和固定 ID。局部同步函数只修改现有节点的文本、属性、隐藏状态和 class，不写入房间根节点的 `innerHTML`。
2. 将语音识别启动、语音识别错误、空回答提示、回答提交中和下一题返回后的 `renderPage` 替换为局部同步。
3. 下一题返回后先同步题号、清空回答和恢复按钮，再异步调用现有题目朗读逻辑；摄像头节点不重新挂载。
4. 如果局部同步时房间节点不存在，则仅在会话仍有效且当前路由正确时降级执行一次完整渲染，以支持首次进入和异常 DOM 恢复。
5. 静态校验要求高频处理函数使用局部同步标记，并禁止在这些函数片段内恢复全页渲染。

## Risks / Trade-offs

- [局部 DOM 与状态不一致] → 所有进行中动态字段统一由一个同步函数更新，避免多处分散操作。
- [阶段切换误用局部更新] → 会话开始成功、面试结束和报告生成继续使用完整渲染。
- [用户正在输入时被覆盖] → 仅在状态明确变化或输入框未聚焦时同步文本；提交下一题时按状态清空。
- [旧异步回调污染新题目] → 继续校验 `panoramaOperationToken`、会话状态和当前路由。
