## Context

`interview-panorama` 页面由 `renderPanoramaInterviewPage` 在准备、进行中、分析中、分析失败、报告和问答记录之间切换。准备页同时承担摄像头权限、无摄像头降级、岗位简历设置和开始操作；进行中页面使用 `syncPanoramaRoomUi` 局部更新题号、计时、朗读、语音和回答状态，以避免整页重绘闪烁。

## Goals / Non-Goals

**Goals:**

- 清晰区分设备状态、隐私边界、面试设置和进入房间操作。
- 使用真实摄像头状态、岗位、已选简历和难度表达准备情况。
- 提升 AI 面试官、用户画面、题目朗读、计时和回答区的沉浸层级。
- 保留媒体生命周期和 `syncPanoramaRoomUi` 局部更新方式，避免恢复闪屏问题。
- 统一真实复盘、分析中和失败状态，并适配窄屏和键盘操作。

**Non-Goals:**

- 不上传、保存或分析视频、画面动作和原始音频。
- 不修改摄像头/麦克风权限申请、语音识别、题目朗读或计时规则。
- 不修改面试题数量、智能体提示词、评分算法或持久化规则。
- 不新增虚拟背景、设备选择器或音色替换。

## Decisions

### 使用独立全景面试页面模式

仅在 `interview-panorama` 路由增加 `panorama-interview-workspace-mode`。样式限定在该模式下，避免影响 AI 文字面试和两类历史记录页。

### 准备页保留单一媒体预览节点

继续使用唯一的 `panoramaCamera` 视频节点和 `attachPanoramaCamera` 绑定流。准备页只增加设备状态条、分区标题和包装类，不复制视频节点，避免媒体流重复播放。

### 设备状态完全来自真实媒体状态

摄像头已连接、请求中、权限拒绝、不可用和无摄像头模式继续由 `panoramaCameraState`、`panoramaStream`、错误与诊断提示决定。页面不推测设备可用性，也不把本地预览描述为已上传。

### 保护进行中的局部更新

`syncPanoramaRoomUi`、`updatePanoramaAnsweringUi`、`syncPanoramaRoomMessage` 和计时函数保持不变。新增视觉包装必须保留现有 ID 与数据属性，不通过整页 `renderPage` 响应每次按钮或语音状态变化。

### 窄屏使用单列沉浸布局

桌面端保留 AI 面试官与用户画面的双列舞台，窄屏改为单列并缩短画面高度；回答操作和设备设置按钮可换行为整行，长提示可自然换行。

## Risks / Trade-offs

- [样式改动再次引入闪烁] → 不改变局部同步函数与媒体节点绑定，只调整结构和 CSS。
- [复制摄像头节点导致流冲突] → 页面每个状态继续仅渲染一个 `panoramaCamera`。
- [设备状态误导用户] → 所有状态文案由真实 `panoramaCameraState` 和 `panoramaStream` 计算。
- [小屏画面过长] → AI 面试官和用户画面在窄屏缩短并单列，操作区保持可达。

## Migration Plan

1. 增加全景面试页面模式和真实设备准备状态。
2. 重构准备页设备检查、设置与入场操作。
3. 优化进行中舞台、回答区、分析状态和报告样式。
4. 更新静态契约与资源版本，完成视觉和构建验证。

回滚仅涉及前端结构、样式和资源版本，不涉及数据迁移。
